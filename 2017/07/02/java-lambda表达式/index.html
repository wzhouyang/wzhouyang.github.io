1.2 函数式接口

java8的lambda是向后兼容的，因此对于只含有一个抽象方法的接口，都可以通过lambda表达式来创建该接口的对象。这种接口被称为函数式接口。

Java API在java.util.function包中定义了许多通用的函数式接口。

可以在任意的函数式接口上标注@FunctionalInterface注解，这样做可以让编译器检查该接口是否只包含一个抽象方法；也可以在Javadoc中包含一条声明，说明这个接口是一个函数式接口。

1.4 构造器引用

构造器引用同方法引用类似，不同的是构造器引用中方法名是new。例如String::new 表示String类的构造器引用。

注意：如果一个接口中定义了一个默认的方法，而另一个父类或者接口中又定义了一个同名的方法，应该选择哪一个那？

  1. 选择父类中的方法。如果一个父类提供了一个默认的方法，那么接口中具有相同名称和参数的默认方法会被忽略。

  2. 接口冲突。如果一个父接口提供了一个默认的方法，而另个接口也提供了一个默认具有相同名称和参数类型的方法（不管该方法是否是默认方法），那么你必须通过覆盖方法来解决冲突。
