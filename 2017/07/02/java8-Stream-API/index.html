java8 Stream API

Jul 2, 2017 | Hits

1.1 Stream概述

java8 提供了stream API对结合进行操作。

Stream特点：

  1. Stream自己不会存储元素。元素可能被存储在底层的集合中，或者根据需求产生。
  2. Stream操作符不会改变源对象。相反，会返回一个持有结果的心的Stream
  3. Stream操作符可能是延迟执行的。

1.2 创建Stream

Java8在Collection接口中新添加了Stream方法，可以将任何集合转化为一个Stream。如果面对一个数组，也可以使用静态的Stream.of方法转换为一个Stream。

也通过Arrays.stream(array, from, to)API将数据的一部分转化为Stream。

如果要创建一个空的Stream则可以通过Stream.empty()方法。

1.3 filter,map和flatMap方法

  1. filter

filter方法的参数是一个Predicate对象，即一个从T到boolean的函数。

  1. map 会对流中每一个元素应用一个函数，并将返回值收集到一个新的流中。
  1. flatMap

map和flatmap区别

如果使用的是map方法，返回的是[ …[‘y’, ‘o’, ‘u’, ‘r’], [‘n’, ‘a’, ‘m’, ‘e’]]

如果使用的是flatMap方法，返回的是[‘y’, ‘o’, ‘u’, ‘r’, ‘n’, ‘a’, ‘m’, ‘e’]

1.4 提取子流和组合流

  1. Stream.limit

Stream.limit(n)会返回一个包含了n个元素的新流（如果原始流的长度小于n则返回原始流）。适用于截取制定长度的流。

产生一个包含100个随机数字的流。

  1. Stream.skip

Stream.skip(n)会丢掉前面n个元素。

  1. Stream.concat

Stream.concat 将两个流连接到一起。注意：第一个流不应该是无限的。

  1. Stream.peek

peek方法产生另一个与原始流具有相同元素的流，但是在每次获取一个元素的时候都会调用，便于调试。

1.5 有状态的转换

前面介绍的流都是无状态的转换。当从一个流中取元素的时候并不依赖前面的元素。还有其他有状态的转换。

  1. distinct

distinct方法会根据原始流中的元素返回一个具有相同顺序，去重的新流。
